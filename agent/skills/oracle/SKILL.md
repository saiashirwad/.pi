# Oracle

Use Oracle when you want a second-opinion advisor for:
- tricky debugging
- review of a commit, diff, or design
- low-risk refactor planning
- regression analysis
- architecture sanity checks

Do not use Oracle for:
- routine edits
- straightforward implementation
- bulk file changes
- tasks where the main goal is to write code fast

How to ask well:
- Say what changed or what broke
- Point to files, commands, commits, or diffs
- State the decision you want help with
- Ask for risks, likely causes, or a low-risk plan

Oracle is advisory and read-only in spirit.
Prefer Oracle for judgment, diagnosis, and recommendations—not doing the work.
